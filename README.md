## Overview
This repository implements a robust, real-time teleoperation system for a robotic dexterous hand (Inspire Hand) using a standard RGB web camera. 

Bridging the gap between raw computer vision outputs and physical hardware actuation is a critical challenge in Embodied AI. Raw 2D keypoint estimations often suffer from spatial variance and high-frequency noise, which can cause severe motor oscillation and hardware damage. This project addresses these issues by designing a customized **Perception-to-Actuation Pipeline** that ensures smooth, safe, and highly responsive robotic control.

## Core Methodologies & Optimizations

Unlike standard CV demos, this system incorporates several control-theoretic optimizations to enable stable real-world deployment:

### 1. Dynamic Spatial Normalization
Raw pixel distances vary drastically depending on the hand's distance from the camera. To achieve depth-invariant control, the system dynamically calculates the **palm size** (distance from the wrist to the middle finger root) as a baseline. All finger extensions are normalized against this baseline, mapping relative ratios to the robotic hand's actuation limits (0-1000).

### 2. Exponential Moving Average (EMA) Filtering
To suppress pixel-level jitter generated by the AI inference model (RTMPose), a weighted moving average ($\alpha = 0.7$) is applied to the mapped kinematic sequence. This ensures trajectory smoothness without sacrificing real-time responsiveness.

### 3. Deadband Control for Motor Protection
High-frequency micro-adjustments can rapidly degrade servo motors. A deadband control logic is introduced to filter out imperceptible noise. Actuation commands are only transmitted over the serial port (~30Hz) when the kinematic delta exceeds a predefined safety threshold, significantly extending the motor lifespan.

## System Architecture

```text
RealTime-Hand-Control/
├── core/
│   ├── processor.py      # Core algorithms: Spatial Normalization, Mapping, EMA
│   └── hardware.py       # Serial communication and hardware protocols
├── configs/
│   └── settings.py       # Hyperparameters, model paths, and deadband thresholds
├── weights/              # Pre-trained OpenMMLab models (RTMDet & RTMPose)
├── docs/                 # Documentation assets (GIFs, architectures)
└── main.py               # Main execution loop
```
1. Environment Setup
Create a conda environment and install the dependencies:
```bash
conda create -n hand_control python=3.8
conda activate hand_control
pip install -r requirements.txt
```
Note: This project relies on the OpenMMLab framework. Ensure mmcv, mmdet, and mmpose are properly installed according to their official documentation.

2. Download Model Weights
Please download the pre-trained weights and place them in the weights/ directory. Refer to [weights/README.md](weights/README.md) for download links.

3. Hardware Configuration
Update the SERIAL_PORT variable in [configs/settings.py](configs/settings.py) to match your local setup

4. Run the System
```bash
python main.py
```
